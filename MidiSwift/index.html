MIDI Parser

A straight forward MIDI file parsing project.

Author
Jeff Behrbaum

This is the visual output so many of us are familiar with when we think of MIDI. This is a screen shot from Garadgeband on iPad, but the general UI is pretty common across various industry software applications. As you progress through the code base - reference back to this image as necessary.

Here, keep some things in mind as you begin your MIDI parsing journey:

  • Each "row" represents a track
  • Each "dot/point" on the track represents an event.
  • Events start with time deltas. ie: Time delta since either the begning of the track or the previous event
  • Not all events represent music - there are essentially meta-events which aren't necessarily visible
  • Events should be processed as soon as the information is available to avoid audio lag

This project uses a command-line Swift application to exercise the code which parses a MIDI file. One consideration is that this code when "processing" an event logs and stores information - this is where you'll want to actaully process the event information.

The primary code to retreive all the information you'll need from the parsed file in an object format is as follows:

										
let midiRecord:MidiRecord = MidiRecord()
let midiReader:MidiReader = MidiReader()
											
guard midiReader.openMidiFile(fileName:filePath) != false else {exit(0)}
											
var midiHeader:MidiRecordHeader = MidiRecordHeader()
var lastIdxRead = midiReader.readMidiRecordHeader(hdr: &midiHeader) + 1
											
midiRecord.Header = midiHeader
let trackInfo:MidiTrack? = nil
											
repeat {
	let trackInfo:MidiTrack? = midiReader.readTrack(startIndex:&lastIdxRead)
											
	if let track = trackInfo {
		midiRecord.appendTrack(track: track)
	}
}while(trackInfo != nil)

Remember, this is straigh Swift - no additional iOS AV libraries or anything else. I developed it this way for two reasons:

  • To provide the most straight forward learning process for developers without any magic 3rd party code
  • So that this code could just as easily be incorporated into your iOS, Mac or any other platform where you are using the Swift language.
